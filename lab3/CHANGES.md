# Изменения в Лабораторной работе №3 (по сравнению с ЛР №1 и №2)

Так как регистрация в GitLab требует привязки иностранной карты, эта лабораторная работа была полностью переделана под **GitHub Actions**.

1. **Добавлен CI/CD Пайплайн**:
   - Внедрен процесс непрерывной интеграции и доставки (CI/CD) с использованием **GitHub Actions** (вместо GitLab CI).
   - Создана структура директорий `.github/workflows/` с файлом `main.yml`, который описывает всю логику пайплайна.

2. **Добавлен этап тестирования (`test`)**:
   - В пайплайне реализован джоб `test`, который запускается всегда в любой ветке. Он проверяет базовую структуру проекта (наличие директорий `dags` и `spark`). Пайплайн упадет, если эти директории потеряются.

3. **Настроены правила сборки образа (`build`)**:
   - Джоб сборки Docker-образа (`docker build`) настроен таким образом, что он пропускается (skipped) для веток, названия которых начинаются с префикса `feature/`.

4. **Настроены правила деплоя (`deploy`)**:
   - Джоб деплоя (`docker-compose up -d`) настроен на автоматическое выполнение исключительно для стабильных веток: `main`, `master` и `develop`. Для остальных веток он не будет выполняться автоматически.

5. **Отказ от локального раннера**:
   - Поскольку используется GitHub Actions, локальный запуск контейнера `gitlab-runner` больше не требуется, так как GitHub предоставляет собственные облачные сервера для выполнения задач CI/CD. Файл `docker-compose.yml` в третьей лабораторной был упразднен.
